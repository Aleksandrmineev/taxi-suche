WhatsApp → База заказов → Страница поиска (MVP)

Ниже — полностью рабочий минимальный стек без сервера:

Экспорт чата WhatsApp → TXT.

Скрипт-парсер (Python) преобразует TXT в data.json (телефон, адрес, дата).

Статическая страница index.html загружает data.json, даёт поиск по цифрам телефона и показывает последние 10 заказов.

1) Экспорт чата WhatsApp

В WhatsApp выберите чат → Ещё → Экспорт чата → Без медиа → сохраните .txt.

Поместите файл рядом со скриптом, например chat.txt.

2) Парсер: parse_whatsapp.py
Базовый запуск: python3 parse_whatsapp.py --input chat.txt --output data.json --config stopnames.json

Извлекает:

phone — нормализованные цифры телефона (оставляем только + и цифры, для поиска используем только цифры),

address — эвристика: текст после ключевых слов (Abholung/abholen/Adresse/von/pickup/откуда/адрес и т.п.),

timestamp — дата/время сообщения (поддержка типичной формы экспорта WhatsApp: DD.MM.YYYY, HH:MM - Name: Message).

После запуска появится data.json.


Как обновлять базу

Экспортируете новый chat.txt.

Запускаете python3 parse_whatsapp.py → получаете обновлённый data.json.

Обновляете data.json на хостинге (или в WordPress Media/темe) — страница сразу подхватит новые данные.

Варианты интеграции

Статический хостинг (GitHub Pages, Netlify): положить index.html + data.json.

WordPress: загрузить data.json в /wp-content/uploads или в тему, а index.html сверстать как кастомную страницу (или шорткод, который отдаёт этот HTML). Можно сделать эндпоинт /wp-json/orders/latest и проксировать туда data.json.

SQLite/REST (при росте): хранить записи в SQLite/PostgreSQL, делать поиск по API (Express, WP REST, etc.).

Что ещё можно улучшить

Более точное выделение адреса (NER/регексы по шаблонам улиц/PLZ в AT).

Авто-нормализация форматов телефонов в E.164.

Слияние дублей (одинаковый телефон + похожий адрес в течение X часов).

Импорт из нескольких чатов, пометка источника.

Ручная правка и валидация через «админ-страницу».

Итог: задача несложная. Уже есть MVP: парсер + страница поиска. Могу допилить под ваш реальный формат чата (с примерами), подключить к WP и настроить регулярное обновление.
